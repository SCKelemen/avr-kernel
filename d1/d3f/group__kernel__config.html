Detailed Description

Configuration options for the kernel.

These configuration options allow the kernel to be customized to meet usage needs. The number of usable threads may be configured, along with the stack size for each thread. Stack canary values may also be enable or disabled for the kernel.

Macro Definition Documentation

#define F_CPU

The clock speed of the MCU as used by _delay_ms and so on.

#define KERNEL_USE_ASSERT

If defined, KERNEL_USE_ASSERT enables assertion checks within the kernel. When an assertion fails, kn_assertion_failure is called.

#define KERNEL_USE_STACK_CANARY

If KERNEL_USE_STACK_CANARY is defined, the kernel will place a canary value at the top of each thread's stack, which it uses to determine if the thread has had a stack overflow.

#define MAX_THREADS

The maximum number of threads that may be enabled simultaneously. Valid thread id's to use will be from THREAD0 to THREAD[MAX_THREADS - 1]. Value must be in the range [1,8].

#define STACK_CANARY

The canary value used if KERNEL_USE_STACK_CANARY is defined. Must be a 1 byte value.
