#define INITIAL_STACK_USAGE

The amount of space used when a stack is set up for a new thread.

Space requirements are: entry point address (2 bytes), thread parameters (thread_id and pointer; 3 bytes), bootstrap function address (2 bytes), 18 callee save registers (18 bytes). The callee save registers are necessary because a thread is entered by returning from the scheduler in the same manner as yielding.

See kn_create_thread for the behavior of this function. kn_create_thread is a wrapper implemented that sets up the stack to avoid stack corruption when a thread replaces itself.

Warning
As with kn_create_thread, this function does not return if t_id is the currently active thread.
void kn_init ( )
static

Provides initialization of the kernel. Is automatically called in the .init8 section, just before main() is called. When the program enters main() the kernel is running with only THREAD0 active. The user must enable interrupts before using any of the timing features of the kernel.

Warning
Due to the way that the linker processes symbols, this function will only be inserted in the .init8 section if another function from this source file is used.
If for any reason you try to manually call this function after main() has been called, you'll totally break your program...
void kn_scheduler ( )

Enters the scheduler without saving the state of the calling thread. Does not return. Should only be used when a thread has been disabled or it is being replaced.

void kn_thread_bootstrap ( )

Pops a thread's parameters off of the stack and into the correct registers, then jumps to the new thread.
